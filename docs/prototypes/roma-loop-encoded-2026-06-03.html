Why this removes every place a false "done" can hide

Five mechanics, in encode order

1

The tree ledger makes coverage total

The vision is enumerated as nodes — an internal node's greenness is derived (green only when all children are green), so you cannot mark a parent done while a child is red. The ledger persists in brain_meta under one JSON key, so the frontier survives across sessions. Nothing in the tree is "off the books."

2

Leaves are too small to fake

When the unit of work is a single predicate, "skip it" stops being a cheap continuation. An unmet leaf is a concrete red predicate, not a smaller, plausible version of success. decompose() refuses to split a green node, so re-decomposition only ever happens on real failure — the loop terminates.

3

The anti-laziness gate fires pre-green

Before a leaf is even handed to the court, the existing Stop-hook reads the trace and, on a premature stop, emits {"decision":"block","reason":...} so the harness refuses to stop and feeds the reason back. This is the Dr. MAMR rule made operational: never reward "short," always reward "verified-complete." The optimizer's incentive to cut corners is intercepted before the court is ever consulted.

4

The court must refute on the real artifact — the worker never judges itself

The load-bearing rule. The agent_id recorded at claim_leaf is held apart from the verdict authority. The court runs a deterministic, anti-tamper gate against the actual artifact — py_compile, the pytest suite, or a live-DOM CDP check — mirroring the existing reflexion deterministic-validator pattern. "I think I'm done" becomes "the court couldn't break it." Checks live outside the executor's reach, because the tampering is real.

5

Loop-until-dry; the founder is the only escape

Red leaves re-decompose and re-enter; the undone frontier only shrinks; sweep() reports done only on a full green sweep. The single honest exit is needs_root — decomposition floor hit, or a tie the jury can't rule. The vision, the taste, what "good" feels like stays the founder's call. A garbage requirement would verify perfectly into a garbage product, so the right requirement is never delegated.

A vision, end to end

Root → leaves → green sweep, narrated

Reading the loop as a single trajectory — the path a real vision takes from the founder's head to a verified-complete sweep.

root

The founder states a vision. create_root() writes it as the root node (open), persisted in the tree ledger. This is the only point where taste enters — everything below is mechanical.

decompose

decompose() splits the root into children, and splits those again — never simplifying, only subdividing — until each leaf is one machine-checkable predicate with a gate_kind. Internal nodes hold no predicate; their greenness is derived from children.

claim

The Workflow fans the executor fleet onto open leaves. Each executor claim_leaf(agent_id) — leaf flips to claimed, owner recorded. Many leaves run in parallel; none may certify its own work.

pre-green gate

When an executor signals "done," the anti-laziness gate runs first. A premature stop is blocked with a reason fed back to the agent. Only work that survives the never-reward-short guard is allowed to reach the court.

court

The court harness runs the real gate on the actual artifact — compile it, run the test, drive the live DOM. The leaf turns green only if the court fails to refute it. If it refutes, the leaf goes red and re-decomposes back into the tree.

green sweep

Green flows upward; a parent greens only when all its children are green; the verified leaf's lesson promotes into the compounding skill library. The vision is done only when a full sweep() shows every leaf green. Anything the jury can't rule, or any floor the tree can't bottom out, surfaces to the founder as needs_root — never a faked green.

Safety contract (load-bearing)

Encoded as additive artifacts only

The encode never mutates what already works. Every claim below is part of the contract the method was approved under.

  • Additive only. New modules requirement_tree.py and court_harness.py; the server.py change is at most one added line in build_server plus a new import — zero existing handlers touched (all 40 @mcp.tool handlers stay as-is).
  • Persistence rides existing storage. The tree lives as one JSON doc under a new brain_meta key via set_meta/get_meta — which already do INSERT … ON CONFLICT(key) DO UPDATE. No new table, no schema migration; never touches fragments/skills/-wal/-shm.
  • The live DB is never mutated. The daemon is down; this is code that compiles and imports cleanly — no run against brain.db.
  • The court reuses proven patterns. The deterministic-validator shape (reflexion) and the live-DOM CDP gate (_verify_live_now.py) are mirrored, not reinvented; the anti-laziness Stop-hook is wired in, not rewritten.
  • .githooks is untouched. Secrets stay op:// references — never inlined.
The way to finish a vision is not to shrink it — it is to fracture it into externally-verified atoms and refuse to let any exit on the worker's say-so.

This document encodes the method the founder approved: the tree ledger in the brain holds total coverage; the anti-laziness gate fires pre-green so "short" is never rewarded; the external anti-tamper court must fail to refute the real artifact before green; the loop runs until dry; done is a full green sweep. The one thing the loop cannot do is be the founder — choose the vision, judge the taste, rule when the jury ties. Everything else is mechanical, additive, and grounded in code that already exists.
