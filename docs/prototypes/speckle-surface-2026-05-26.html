Why this exists

"i also clearly remember that i signed off a fucking speckle for adaptation for the wiring and data spine · what happened to that · where is that?"
— founder, 2026-05-26

The Speckle data spine IS shipped (AgDR-0012 + AgDR-0016 + AgDR-0017). Every wire is a Speckle Operations.send / receive segment with DiskTransport default at .speckle/<project>/. 3 SHARE primitives exist (share.server · share.publish · share.subscribe). The ADAPTER category exists. The problem: it's invisible by design (founder original ask: "no server, no Docker, fully offline"). Now the founder wants to see the spine — without surrendering the invisible-by-default behavior. This prototype surfaces Speckle through four lightweight UI signals that DO NOT add cognitive overhead for users who don't care.

Surface 6 · Brain × Speckle wiring ⚑ FOUNDER ADDED 2026-05-26

Founder ask: "figure out how this connects with the brain system." The brain (personal-brain-mcp, AgDR-0044) owns scoped memory (USER / PROJECT / FIRM) + facts + skills. Speckle owns per-wire Version history + DiskTransport state per project. Today they don't know about each other. Six wiring points unify them without surrendering brain's scope discipline or Speckle's invisible-by-default behavior.

W1 · Unified project identity

brain._infer_project_id(cwd, git_remote) already exists (personal-brain-mcp/src/personal_brain/server.py). Speckle's DiskTransport writes to .speckle/<project>/. Unify: pass brain.context.project_id into speckle_wire.set_project() so both systems agree on what "project" means. No drift. No double-prompting the user "which project?".

W2 · Speckle Version → brain fact (write-through)

Every successful Operations.send on a wire (in speckle_wire.py) fires brain.write([{op:"ADD", kind:"fact", scope:"project", payload:{wire_id, version, sender, count, ts}}]). Brain becomes the queryable index over Speckle history. Cost: one cheap MCP call per wire commit. Failure-tolerant via ResilientBrainClient circuit breaker.

W3 · brain.context pre-injects current wire pins

When the founder types in Composer, brain.context (already fires per UserPromptSubmit hook in ~/.claude/settings.json) returns recent wire-version facts for the active project. The LLM sees <brain_context>walls v0024 · doors v0017 · slabs v0009</brain_context> silently. No prompt change. The LLM can write receive("walls", "v0024") with grounded knowledge.

W4 · Skill mint captures Speckle Version pins

On graph cook success, brain.skill_mint(trace) is already called via the Stop hook. Extend the trace to include speckle_pins: {wire_id: version} for every wire the cook touched. A minted skill is deterministically replayable: next time it runs, it receive()s the exact pinned Versions, not "latest." This is what makes a workflow a workflow vs. a one-shot.

W5 · Speckle panel gains BRAIN MEMORY section

The panel (Surface 4 above) gets a new section under HISTORY:

W6 · Cross-machine SHARE auto-config via brain wiring_announce

When a share.server node lazy-starts a local Speckle server, it calls brain.wiring_announce({mcps:{"speckle_local":"http://<host>:<port>"}}). Other machines in the same FIRM scope receive that endpoint in their brain.context and can share.subscribe with one click instead of manual URL paste. No central registry; brain IS the registry per AgDR-0044.

Forks for signoff — REVISED 2026-05-26 after founder narrowing

Founder pre-approved 2026-05-26: F4 (footer pill format A · spk ● disk · p-664 · v0024), F5 (panel modal A · matches Memory Explorer + ai.plan History contracts). F1/F2/F3 are withdrawn with the surfaces above. New brain-wiring forks below.

ForkPickTrade-off
FB1 — W2 brain write-through cadence

Real-time = perfect brain knowledge but ~10/s MCP calls during a heavy cook. Batch is 80% of the value at 5% of cost. Hourly loses session-grain queries.
FB2 — W3 brain.context wire-pin verbosity in pre_prompt

LLM token cost — A burns ~80 tok/wire, B ~12 tok/wire, C ~3 tok/wire. B is the sweet spot: LLM gets enough context to reference without bloating the prompt.
FB3 — W4 skill replay strictness when a pinned Version is missing

STRICT preserves the contract that "minted skill = deterministic." WARN drifts silently. ASK is correct but interrupts flow. Recommend A; offer "rebuild pins" affordance in the modal.
FB4 — W6 firm-scoped MCP announce

AUTO is frictionless but leaks per-machine endpoints across the firm. OPT-IN preserves user agency (USER-AGENCY MANDATE). NEVER drops the W6 value.
FB5 — Footer pill click action

A is consistent (one click → full panel). B optimizes for the memory-first workflow but adds a second click for non-brain users. C is lightweight but adds a new UI pattern.

What this prototype does NOT add

  • No new wire transport. DiskTransport stays default per AgDR-0016.
  • No mandatory account. Server stays opt-in via SHARE nodes.
  • No new dependency. All UI hooks read from `app/speckle_wire.py` state already exposed.
  • No prototype-breaking change. All other shipped surfaces (canvas, composer, footer items) preserved.
