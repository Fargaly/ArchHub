Critical accessibility gap

⚠ CRITICAL — Procurement blocker
0 aria-* attributes across 8 596 lines

Every interactive button, input, modal, dropdown, node-on-canvas — none has an aria-label. Screen readers can't announce anything. Keyboard navigation has no visible focus ring. Enterprise AEC procurement expects WCAG 2.1 AA.

11 occurrences of role / tabIndex / onKeyDown exist (some keyboard work), but no consistent floor across the codebase.

Before / after a typical inline style

BEFORE — magic numbers
<div style={{
  padding: 14,
  fontSize: 9.5,
  borderRadius: 5,
  boxShadow: '0 4px 12px rgba(0,0,0,0.4)',
  transition: 'opacity 0.2s ease',
}}>

padding:14 not in any scale
fontSize:9.5 half-pixel, blurry on non-Retina
borderRadius:5 not in any scale
boxShadow handwritten, inconsistent with other components
transition unique easing, no shared vocab

AFTER — token-driven
<div style={{
  padding: LM.size[3],
  fontSize: LM.font.xs,
  borderRadius: LM.radius.sm,
  boxShadow: LM.shadow.s2,
  transition: `opacity ${LM.motion.base}`,
}}>

size[3] = 12px · in scale
font.xs = 10px · in scale, no half-px
radius.sm = 5px · in scale
shadow.s2 = panel-elevation tier
motion.base = 200ms cubic-bezier · shared

Open forks for founder

F-1 Scale-step base unit. Token 1 uses a 4-px base. Material uses 4, Apple uses 8. Pick 4 (denser) or 8 (cleaner)?
F-2 Font scale max step. Today's largest font is 22px; the scale tops at xl: 22. Add xxl: 28 / xxxl: 36 for Composer's hero state?
F-3 LM.brand.notion defaulted to LM.ink (greyscale). OK or specific color?
F-4 Phase 3 migration cadence — 1–2 weeks of refactor with no new features. Serial (block features) or parallel to M3?
F-5 Accessibility scope — full WCAG 2.1 AA, or only the AAA-blocker subset (focus rings + aria-labels + keyboard nav)?
