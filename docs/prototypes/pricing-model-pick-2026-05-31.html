Model A — pay per message

+Simple to grasp: a tier = a monthly message bucket.
+You host the LLM on Studio/Firm — a real margin to capture.
+Firm tier ($299) anchors a real enterprise price.
−Hosting the LLM = you fund the upstream API bills (real spend + the proxy must be turned on).
−Message quotas can feel arbitrary to a buyer ("how many is 2,000?").

Model B — pay per seat

+BYO-key default = you carry almost no LLM cost; near-pure-margin.
+Per-seat is the SaaS norm teams expect — easy to forecast.
+3-seat minimum pushes teams, not just solos.
−BYO-key means each user wrangles their own API key (friction).
−No hosted-LLM upsell = you leave the inference margin on the table.
My read (you decide): If you want to launch fast + cheap and avoid funding LLM bills, Model B (per-seat, BYO-key) is the lower-risk start — near-zero cost-of-goods, the proxy stays off, teams pay per seat. If the bet is that hosting the AI is the product (the brain + your own models — your north star), Model A sets you up to capture that inference margin, but you fund the API balances + flip the paid proxy on.
