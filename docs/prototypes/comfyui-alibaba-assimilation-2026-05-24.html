Render workflows we never touched — Alibaba models, ComfyUI graphs, Revit / Rhino assimilation

You asked: cheap Alibaba models + ComfyUI community workflows + creative assimilation. Revit viewport to AI render? Photo to Rhino mass? Yes — all feasible. Here's the strategy.

The big finding

ComfyUI workflows are graph-as-JSON. ArchHub workflows are graph-as-JSON (AgDR-0040). They are the same substrate. Assimilation is not "rebuild ComfyUI" — it's one connector + a workflow importer that wraps any ComfyUI graph as a Capability Node. Alibaba via DashScope adds 3–10× cheaper LLM + the cheapest image-to-video on the market ($0.035 / clip vs $0.50+ Runway/Pika).

part 2 · ComfyUI ecosystem

ComfyUI patterns that map straight onto ArchHub

Graph-as-JSON comfyui core

Every ComfyUI workflow is a JSON document — nodes, edges, params. Server runs headless by default: POST /prompt with the JSON, GET /view for outputs. Identical mental model to ArchHub's workflows/graph.py + runner.py. Translation is a 200-line adapter, not a rewrite.

ControlNet Depth / Canny archviz

SDXL + Depth ControlNet preserves spatial relationships — the exact thing Revit viewport renders need. Feed a depth map from Revit (we already have it via .NET API), AI fills in materials / lighting / context. Stage 1 = SDXL, Stage 2 = upscale.

Hunyuan3D-2 tencent · OSS

Single image → textured 3D mesh in 4 stages: bg-removal → geometry → optimize → texture. 1.1B params, runs on a 24 GB GPU locally OR via ComfyUI cloud node. Output = .glb / .obj → direct Rhino import.

Stable Fast 3D / TripoSR stability + tripo

Single-image 3D mesh in under 1 second on a decent GPU. Lower fidelity than Hunyuan but fast enough for iterative "drop a photo → mass model" UX. Both wrap into comfyui.image_to_mesh.

ComfyUI-3D-Pack community

Single repo bundling Hunyuan3D + StableFast3D + InstantMesh + CRM + TripoSR + 3DGS + NeRF nodes. One install → 7 mesh-gen pipelines exposed as ComfyUI nodes. One ArchHub Capability Node per pipeline.

ComfyUI Manager 2000+ nodes

Community node registry — 2000+ custom nodes one-click installable. IPAdapter, AnimateDiff, Impact Pack (face-detect), Frame Interpolation, SAM segmentation. ArchHub's library_search → library_create contract maps 1:1 onto the registry — point Composer at it and it becomes a discoverable node source.

part 3 · integration strategy

Three tiers — pick how deep we go

Each tier compatible with the previous. Tier 0 ships in days, Tier 2 in weeks.

Tier 0 · Connector wrapper

connector.comfyui · connector.dashscope

~3 days · 2 PRs

Two new entries in app/connectors/. comfyui talks to a local or remote ComfyUI server via its REST API (POST /prompt + GET /view). dashscope wraps Qwen / Wan / Qwen-VL endpoints. Both register as standard ArchHub connector ops — appear in tool_engine immediately.

  • +24 connector ops (12 per connector)
  • Works against user's own ComfyUI install
  • BYO DashScope key (Solo plan)
  • Zero new node code — uses connector.run machinery
Tier 1 · Workflow importer

library.import_comfyui_workflow

~7 days · 1 AgDR + 1 PR

Paste a ComfyUI JSON URL or upload the file → ArchHub creates a Capability Node wrapping that workflow, auto-derives typed I/O ports from the workflow's external inputs/outputs (same pattern as AgDR-0040 slice 3). The community's 2000+ workflows become 2000+ ArchHub nodes on demand.

  • 1-click adopt any community workflow
  • Auto-derived inputs (any unconnected text/image input port)
  • Saved to Library, queryable via node_search
  • Versioned via Speckle (every import is a Version)
Tier 2 · Deep wrap + render primitives

render.* · vision.* · mesh.* node families

~14 days · 3 AgDRs + 4 PRs

First-class typed nodes for the 4 use cases below, each composing the connector / importer above. render.archviz, render.upscale, vision.subject_describe, mesh.from_image, mesh.import_to_host. These are what the Composer's node_search finds first when the user says "make this Revit view look real."

  • Inline render preview node (no canvas switch)
  • Cost-aware routing (Wan-fast vs Wan-quality)
  • Host-aware (Revit / Rhino / Max emit + ingest the right format)
  • Verified Skill: "Hero render from Revit view" — installable
~3d / ~10d / ~24d cumulative · AgDR per tier
