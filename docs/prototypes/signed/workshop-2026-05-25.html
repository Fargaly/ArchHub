7 new mandates · drafted by Hat 6

① Definition of Shipped

"Shipped" = founder can launch ArchHub + click around for 60s + SEE the thing working. User-visible path. 60-second discoverability. CDP screenshot proof. Real interaction. Tests passing ≠ shipped.

② Prototype-Is-Contract

Signed prototype IS the spec. Shipped JSX mirrors it 1:1. Prototype moves to prototypes/signed/, read-only post-signoff. Visual diff before claiming parity.

③ No Open Threads

Loop ends with ZERO open threads. Banned: TODO(founder) · "test later" · "ship X first". Per-iteration grep gate. Tasks tagged "founder verifies" are forbidden.

④ Pre-flight Check

7-question Y/N gate before "shipped" word allowed. Built? Restarted? Reachable? Visible? Clickable? Persistent? Discoverable? Output pasted verbatim into report.

⑤ Post-loop Audit

Script tools/loop_audit.ps1. git log · diff stat · TODO grep · PID check · CDP bundle hash · proof screenshots · AgDR linkage. Audit BLOCKS "done" report on any fail.

⑥ Rollback Protocol

Founder finds gap → no TODO, no apology · acknowledge in one line · re-run preflight live · finish OR git revert in SAME response · append docs/FAILURE_LOG.md · no new work until gap closed.

⑦ Workshop Gate

STOP shipping + convene workshop on: spec ambiguity · cross-surface change (≥3 of jsx/bridge/tool_engine/connector/runner) · founder-frustration signal · repeat regression · preflight fails 2× · loop with zero verified ships.
