Host node — three design directions

Concrete prototypes for the Revit host node carrying its 13 operations. Each direction is opinionated — they trade off canvas density vs panel power differently. Built from grounded research into ComfyUI, n8n, Blender Geometry Nodes, Unreal Blueprints, and Houdini.

What every direction shares. Every op visible somehow (no hidden dropdown). Every parameter promotable to a socket. Right panel has tabs, search, sliders. Floating settings bar / right-click for run-mode (no buttons cluttering the node body). All three are renders of the same Revit project (live-A203.rvt) at the same moment so the comparison is honest.

DIRECTION A Op grid + Houdini flags

All 13 ops live as a tile grid on the node face. Active op expands inline. Per-op coloured flags carry run state (no buttons). Right panel = tabs + search across every op. The most assertive answer to "nothing hidden."

Borrows from: ComfyUI Convert Widget to Input · UE Split Struct Pin · Houdini flags · n8n displayOptions + NDV tabs · n8n Pin Data.
What's easier:
  • Op discovery is instant — all 13 visible.
  • "Nothing hidden" honoured: every op + its state is on the node face.
  • Flag system scales: a 14th op is one more tile.
  • Promote-to-socket is one click and visible at left rail.
What's harder:
  • Node grows tall when an op is expanded — collisions with other nodes if dense graph.
  • Left socket rail can stack heavily if user promotes many params.
  • Tile-expansion animation has to feel deterministic, not jumpy.
DIRECTION B Stacked widgets + Blender dual surface

Single tall node. Every op = a section. Every parameter = a widget row. Each row promotable to a socket via a single drag. The right panel is a 1:1 mirror of the node body (Blender Group Input ↔ modifier-panel model). Most ComfyUI-native; cleanest gesture model.

Borrows from: ComfyUI widget stack + Convert Widget to Input · Blender Geometry Nodes Group-Input ↔ modifier-panel dual surface · UE Split Struct Pin (section-as-struct) · n8n Pin Data per op.
What's easier:
  • One mental model: the node IS the form, the panel is a magnifier.
  • Promote-to-socket = single drag on the row's left-edge dot.
  • Mirror means you can edit either surface — Blender's GN modifier slider works identically.
What's harder:
  • Node is TALL — ~780 px even with most sections collapsed. Canvas scroll friction.
  • Visual hierarchy between section headers and rows must be ruthless or it becomes a wall of text.
  • Mirror sync needs zero-latency feedback both ways.
DIRECTION C Houdini multi-tab + channel-ref soft wires

Compact node face. Tab strip across the header — one tab per op + Common + All. Active tab shows its params inline; right panel is a full Houdini-style param editor (tabs + search + colour-coded states). Channel-reference / soft wires keep the canvas clean even when many values are linked.

Borrows from: Houdini param editor (tabs, search, colour-coded states, spare parameters) · Houdini flags · Houdini channel references (ch()) · ComfyUI floating selection toolbar · UE Promote to Variable.
What's easier:
  • Most compact node face of the three — scales to 30+ ops.
  • Channel references = no noodle spaghetti for shared values; hover-overlay still discoverable.
  • Power-user surface (Spare Parameters, expression toggle) without burdening newcomers.
  • Floating settings bar = no buttons on the node body (founder constraint).
What's harder:
  • Strict reading of "nothing hidden" — non-active tabs hide their params. Reframe needed: tabs are not dropdowns.
  • Three promotion gestures (socket / channel-ref / variable) is power but adds learning.
  • Channel-ref discoverability needs hover-overlay polish.

Pick a direction (or steer me)

Once you point at one (or describe a hybrid you want), I write the revised AgDR-0012 against the founder mandate, then start the ReactFlow migration with this host-node design as the first React component on the new foundation.

  1. A — Op grid + Houdini flags · most assertive answer to "nothing hidden"; newcomer-friendly; node grows when ops expand.
  2. B — Stacked widgets + Blender dual surface · simplest gesture model; tall node; the node IS the form.
  3. C — Houdini multi-tab + channel-ref · most professional; scales furthest; tabs are the "non-dropdown dropdown".
  4. Hybrid — tell me which pieces of each you want fused.
