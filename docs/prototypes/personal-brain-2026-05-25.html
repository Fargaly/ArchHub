Planning council

Seven-hat verdict

Each hat read the same R&D pack — 6 parallel scouts hit 50+ projects across agent memory, auto-skill mining, sync substrates, neuroscience, MCP ecosystem, and ArchHub's own code. Verdicts compressed below.

Architect
Build it as a two-tier brain on a shared substrate. Fast episodic write (hot path, <200ms) + slow semantic consolidation (background worker). One MCP server, two transports (stdio + streamable HTTP). FastMCP Python. Anchored to ArchHub's existing app/memory/graph.py.
  • Layer 5 hook inserts at llm_router.py:1366 (pre-stream) + :1402 (post-tool) + :1378 (turn-end)
  • Reuse LibraryGate + TurnState patterns — Layer 5 is a sibling of Layer 3
  • Loro CRDT for memory graph sync; OR-Set for skill index; PN-Counter for use-counts
  • HLC timestamps (Lamport+wallclock, 64-bit) for cross-device ordering without clock sync
Skeptic
Three landmines literature confirmed. Auto-mint without quality gate = library poisoning within ~100 skills. CRDT merges hide semantic duplicates. Echo Trap collapses exploration if you reward skill-reuse blindly.
  • SkillReducer paper: 26.4% of real-world skills lack descriptions; auto-miner reproduces this
  • RAGEN "Echo Trap" — reward variance cliff when agent self-relies on past trajectories
  • CRDT structural merge ≠ semantic dedupe — two devices mint same skill, both keep
  • ACL revocation in CRDT world is hard — BeeKEM only solves forward, not retroactive
  • Mem0's own 2026 retrospective: 25% accuracy drop scaling 1M→10M tokens
User (founder)
"Always reading my prompt and finding relative skills" = real-time semantic skill retrieval on UserPromptSubmit. Works fully only in Claude Code + ArchHub. ChatGPT/Codex/Gemini need an MCP prompt primitive the user invokes (e.g. /brain) OR a tool the model is taught to call first.
  • Universal silent injection IS NOT possible today across all clients — be honest about it
  • Claude Code + ArchHub Composer = full 5-loop substrate (silent context inject)
  • Cursor = partial (per-tool-call hook, no prompt-level)
  • ChatGPT/Codex/Gemini/Cline/Continue = server-instructions on session init + user-invoked prompts
  • Skill mint always works — driven by Stop hook in clients that have one, by post-turn scan elsewhere
Security
Secrets NEVER in the brain. Brain stores references only (op://vault/…). PreToolUse hook resolves via 1Password CLI / Infisical / Windows Credential Manager at call time, scrubs trace logs. Bipartite ACL (arXiv 2505.18279) for per-fragment access — user/project/firm/community/global scopes.
  • OAuth 2.1 + PKCE + DCR for remote MCP (mandatory for ChatGPT desktop)
  • age-encrypted per-doc keys derived via HKDF from vault root (Obsidian-pattern)
  • Promote-with-redaction LLM pass before any fragment crosses scope boundary
  • Audit log per fragment access — retrospective check on every read
  • Stdio = env-var bearer; HTTP = OAuth — never mix
R&D Scout
Buy 80%, build the 20% that matters. Closest off-the-shelf substrate: agentmemory (9.3k stars, 51 MCP tools, 12 hooks, P2P mesh sync, 95.2% LongMemEval). Adopt its hook surface; bolt on bipartite ACL + redaction + ArchHub graph as the user-side layer.
  • Voyager pattern: docstring-as-key indexing, top-5 retrieval, GPT-as-critic gate
  • SkillWeaver pattern: hone (N≥3 sandbox trials) before publish
  • Anthropic Claude Skills budget: 1% of context, least-invoked drops first when overflow
  • Generative Agents score: α·recency + β·importance + γ·relevance
  • ExpeL: add/edit/upvote/downvote insight curation (skills as wiki, not append-only)
Pragmatist
Three phases. Phase A (2 weeks): local-only MCP server, Claude Code + ArchHub wired. Phase B (1 week): Fly.io multi-device sync. Phase C (3 weeks): ACL + redaction + federation. Total: ~6 weeks for V1. Phase A alone gets 70% of the user value.
  • Reuse app/memory/graph.py (505 LoC) — already SQLite + nodes/edges shipped via AgDR-0042
  • Reuse app/memory/sync.py (267 LoC) — push/pull/merge already works; swap JsonFileTransport for Loro
  • Reuse app/library_gate.py pattern (224 LoC) — MemoryGate is its sibling
  • New code: ~1500 LoC for MCP server + embeddings + reflexion worker + redaction
  • Don't ship community tier in Phase A — moat, not table-stakes
Visionary
Two-year horizon: substrate becomes the network effect. ArchHub differentiates not on "we have AI" but on "your AI gets smarter the more you and your firm use it, and your skills travel with you between Claude / GPT / Cursor / desktop / mobile."
  • Year 1: founder + early firms — local brain + firm sync
  • Year 2: community marketplace — signed skill exchanges between firms (Wikidata-style reputation gates)
  • Year 2+: brain becomes a portable layer (your skills work in ANY AI app that speaks MCP)
  • Macro: the moat is the consolidated graph and the network effect on skill mints, not the LLM
  • Risk: Anthropic, OpenAI ship their own — but they won't go cross-provider. You will.

Decisions blocking Slice 1

Four open forks — founder pick

Per WORKSHOP-GATE mandate (CLAUDE.md), this is cross-surface (touches app/memory, llm_router, tool_engine, cloud_backend). Needs founder sign-off on these forks before code lands.

Fork 1 — Substrate origin
Buy or build the base?
A · Fork agentmemory + bolt on ArchHub layer9.3k stars, 51 MCP tools, 12 hooks, P2P mesh sync, 95.2% LongMemEval already. Inherit the substrate; build the ACL + redaction + reflexion-curriculum tier on top. ~6 weeks → V1.
B · Build from scratch on app/memory/graph.pyFull ownership, no Apache-2.0 attribution constraint, brain shape exactly matches ArchHub's MemoryGraph. ~10 weeks → V1.
C · Adopt Mem0 as managed serviceFastest to demo (~1 day setup), but vendor lock-in, no on-prem option for firms, no auto-skill-mint.
Fork 2 — Skill-mint trigger
When does the reflexion worker decide a trace becomes a skill?
A · Voyager + SkillWeaver hybridCritic-LLM verifies trace succeeded → propose skill → 3 sandbox trials honing → publish only if ≥2/3 succeed. Highest quality, slower (~30s post-turn).
B · Pure trace-similarityHash the tool-call sequence; if ≥5 occurrences in history with success → auto-mint. Fast (<1s), but late-mints.
C · Founder approval-gatedBrain proposes skill; founder one-click approves in side panel. Highest quality control, lowest velocity, abandonment risk.
Fork 3 — Sync substrate
What carries memory + skills across your devices?
A · Loro CRDT + Tailscale meshMovable-tree CRDT fits the memory graph natively; Tailscale removes "where do I host the relay." Best ergonomics, no SaaS dependency.
B · Speckle Versions adapterAlready shipped in AgDR-0042. Content-addressed, immutable, deterministic. But: no live offline; push/pull only.
C · Custom HTTP + ts-wins (current)Stays in ArchHub's existing sync.py code. Brittle for multi-agent; defer the upgrade until V2.
Fork 4 — Community tier timing
Ship cross-firm community sharing in V1 or defer?
B · Defer to V2Federation needs FICAL pattern + DP noise + reputation gating; ~3-4 weeks alone. 90% of user value lives in personal+firm tiers. Ship those first.
A · Include in V1Builds the moat now. But: 10 extra weeks before any seat sees value; risk of getting attack surface wrong on first public release.
