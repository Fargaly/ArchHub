Composer + Speckle wires — the recommended architecture

Grounded prototype of Direction X (with one Direction-Y concession): Composer is the primary IDE, canvas is the execution + inspection surface, every wire is a Speckle Model/Version segment via local SQLiteTransport. ai.plan survives as a real canvas node so chat turns persist as auditable artefacts.

The hard facts that make this work. Speckle is Apache-2.0. specklepy ships MemoryTransport + SQLiteTransport + Operations.send/receive — meaning the wire works with ZERO servers, ZERO Docker, ZERO accounts. The full Base graph, units conversion, decomposition, chunking, content-addressing — all client-side. Speckle Server becomes an OPTIONAL remote transport for collaboration.

Why this satisfies the founder's framing

Founder askHow Direction X delivers
"wire IS a Speckle stream segment" Every wire is Operations.send → hash → Operations.receive. Default SQLiteTransport. Speckle Server is opt-in only.
"composer is the primary IDE" Composer panel left, full height. Tool-use deltas stream nodes onto the canvas. Plan / Auto / YOLO modes. Canvas remains as audit/inspection.
"all three previously-discussed paths" P1 → composer turns are first-class; ai.plan node persists each turn. P3 → adapter ops collapse into per-host send_to_speckle / receive_from_speckle nodes. P2 → glue.script survives as the explicit "Speckle can't carry this" escape.
"AI should compose ops, build new ones" Composer streams graph.create_node / graph.connect / graph.set_param tool calls. node_smith + custom_nodes (already in repo) materialise new ops as typed Python on demand.
"can I stream CAD lines to Revit using THESE ops?" Yes — through Speckle. The Revit + AutoCAD Speckle connectors already implement ConvertToSpeckle / ConvertToNative. ArchHub bundles them; the cross-host glue is the protocol, not the user's problem.
"don't build from scratch" ReactFlow for canvas. Speckle for wire format + serialization + host connectors. Anthropic SDK for tool-use loop. ArchHub builds the binding layer + UX, not the substrate.

Recommendation + order

Direction X with the ai.plan concession. Total 12-16 weeks of focused work.

  1. M1 — Speckle as local wire (1–2 wks). Embed specklepy. Replace ArchHub wire serialization with Operations.send/receive. Default SQLiteTransport.
  2. M2 — First host connector (3–5 wks). Revit first (biggest user base). Bundle Speckle Revit connector or IPC. Two nodes: revit.send_to_speckle + revit.receive_from_speckle.
  3. M3 — Composer v1 (6–8 wks). Qt panel left. Anthropic tool-use loop. Tools: graph.*, speckle.*, node.inspect. Stream input_json_delta events to canvas. Plan / Auto / YOLO modes. Per-project ARCHHUB.md as persistent memory.
  4. M4 — ai.plan canvas node (9–10 wks). Real node carrying prompt + model. Outputs are typed Bases the agent emitted as tool_use results.
  5. M5 — Cross-host wire AutoCAD → Revit (11–12 wks). Validate units, layer mapping, DirectShape fallback. Document the lossiness on hover.
  6. M6 — Optional ServerTransport (13–14 wks). Cloud Speckle as collaborator. "Publish this wire" makes a remote Model. Speckle Automate webhooks for live cross-machine wires.

First valuable shippable: M1 + M2 (5 wks). Closes the litmus scenario end-to-end with a single human-driven canvas + Speckle wires. The Composer comes after.

Risks (honest)

  • DirectShape fallback erodes Revit roundtrip fidelity. Speckle-documented, unfixable without re-implementing parametric semantics. Mitigate: "Receive Blocks as Families" setting; document loss on hover.
  • Speckle's Object model evolves under us (Stream→Project rename happened in 2024). Pin specklepy and connector minor versions; track upstream in CI.
  • Composer is slow on long graphs. Stream is fine for 5 nodes, perceptibly slow at 100. Use Anthropic parallel tool calls + graph.batch_create_nodes + prompt-caching for project context.
  • Local SQLite cache grows shared across Speckle apps. Ship a .speckle/ project-local DiskTransport so cache is scoped per project, not globally.
  • Composer hallucinates node types. Bias the system prompt toward calling graph.list_node_types() first. Use Anthropic strict: true tool use.
  • No single-binary Speckle Server if a user opts in to self-host. Lean on app.speckle.systems for 95%; ship a self-host doc page.
