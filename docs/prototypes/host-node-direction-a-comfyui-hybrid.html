Host node — Direction A base, ComfyUI hover-promote refinements

Founder's recall (2026-05-21): "It will be Option A but consider ComfyUI's approach of main sockets + advanced options. Every field can be wired in/out per its type. Sockets toggle on hovering above the socket area. Like ComfyUI." This is the prototype that materialises those constraints — no extrapolation, no wrapping, no shit. Three stages below show the same host node in three live states: default, hover, post-promotion.

What this refines from Direction A. Direction A's op-grid + flags + active-tile-expand remain. ADDED: every parameter row is wireable on hover (ComfyUI Convert-Widget-to-Input mechanic) — left edge for inputs, right edge for outputs, dashed marker on approach, solid coloured socket when promoted. Sections split into MAIN (always-visible primary I/O — trigger / last_out) and ADVANCED (collapsible — every parameter that can be wired). The right panel mirrors the on-node experience for power users.

What this earns:

  • The dashed marker has to feel deterministic — appears at hover, disappears on leave, never flicker.

ComfyUI parallel: promoted inputs migrate to the node's left edge as "slot" sockets — the original widget still works (constant fallback), but the slot now accepts wires of the matching type. Demotion is the inverse gesture. ArchHub adds the OUTPUT-pluck dimension on the right rail: any cooked field (e.g. list_walls[0].id) can be promoted as a typed output without changing the engine contract.
What this earns:
  • The node face stays calm in the default + grows only where the user invested.
  • Rail labels make wire-target intent explicit — list_walls.view is unmistakable.
  • OUTPUT pluck removes the need for a downstream "extract field" node for common patterns.
What this costs:
  • The rail layout has to flex as the user promotes more — three sockets fit nicely; thirty would need scroll.
  • Wire colour rules: in = purple, out = cyan, typed-wire colour overlays per Slice-D palette inside the rail rendering.
