## Patterns observed

- **Centralization.** All call sites route through a single `app/telemetry.py:track_event(name, **props)` wrapper. The wrapper is the only place `posthog.Posthog(...).capture(...)` is invoked.
- **PII chokepoint.** Every props dict is sent through `pii_redactor.redact_dict()` inside the wrapper before reaching the SDK. `redact_dict` strips: Windows + POSIX paths, Anthropic / OpenAI / OpenRouter / Google / GitHub / PostHog / Sentry-DSN tokens, emails, IPv4 addresses, project-folder names wrapped in quotes. It does not strip names, phone numbers, or arbitrary natural-language personal text.
- **Defense in depth.** Every call site additionally wraps the `track_event` call in its own `try/except` — the wrapper is failure-silent, and call sites do not rely on that contract.
- **Async.** PostHog SDK runs in its own background thread (`sync_mode=False`); the calling thread never blocks.
- **Feature flags.** `telemetry.is_feature_enabled(flag, default)` is wired and uses the same `distinct_id` as events. No grep hits for `is_feature_enabled` at call sites in the scanned modules — present as primitive, not yet consumed.
- **Shutdown.** `telemetry.shutdown()` exists for flush-on-exit; whether it is wired into `atexit` from `main.py` was not verified in this pass.
- **Naming.** All five event names are snake_case. Grammar is mixed: `subject_verb` (`app_started`), `subject_adjective` (`provider_blocked`), `subject_verb_phrase` (`telemetry_opted_in`), `noun_noun` (`skill_run`, `user_feedback`). No consistent `object.action` convention.
- **Sentry parity.** `app/sentry_init.py` shares the consent flag (`if consent_state() is not True: return False`), so a single Settings toggle gates both PostHog and Sentry. Sentry's `before_send` additionally drops events when `PYTEST_CURRENT_TEST` is set; the PostHog wrapper has no equivalent guard.

## Hygiene observations (no recommendations)

- **JSX surface emits nothing.** `app/web_ui/studio-lm.jsx` (~5,000 lines, the primary user-facing surface — canvas placement, wiring, run, composer chat, skill save/load) contains zero tracking calls. PostHog/analytics greps over `app/web_ui/` only match inside vendored React-DOM and Babel bundles.
- **Landing site emits nothing.** `landing/index.html` and `landing/security.html` contain no analytics snippet (no PostHog JS, no GA, no Plausible).
- **Cloud backend emits nothing.** `cloud_backend/` processes Stripe + Polar billing webhooks, sign-in flows, magic-link issuance, and per-company quota — none of which emit telemetry.
- **Free-text property on `user_feedback.comment`.** The redactor does not catch natural-language personal data the user may type.
- **No schema version.** `app/telemetry.py` has no `EVENT_SCHEMA_VERSION` constant — property additions or renames cannot be detected by downstream consumers from the event payload alone.
- **`posthog` is unpinned in `app/requirements.txt`** by design (same class as `cryptography`, `specklepy`, `mcp`). Implicit-local / missing-in-CI gap; the wrapper returns `_client = None` when the SDK is absent.
