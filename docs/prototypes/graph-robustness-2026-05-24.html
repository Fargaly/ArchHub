Decision matrix
on delete
  1. Find upstream src port type
  2. Find downstream dst port type
  3. If src == dst → auto-bridge silently
  4. If src is sub-type of dst (e.g. any) → auto-bridge
  5. Else → broken-wire state · open Inspector recovery
recovery offers
  1. Search library for src → dst adapter
  2. Offer to restore deleted
  3. Offer to swap downstream node
Gap · ArchHub already has typed Ports. Add graph.on_node_delete(id) that runs the decision matrix. Adapter search reuses node_search with {in_type, out_type} filter. Broken-wire UI is new (~150 jsx) but the runner already refuses to cook broken wires today — we just surface the reason inline instead of swallowing it.

property 6 · added per founder 2026-05-24

Bypass · skip the node entirely · upstream passes through

Different from freeze. Freeze = node returns its last cached value. Bypass = node is invisible to the runner, upstream input flows straight to downstream output (port-type matched). No cache, no execute. Same as Photoshop "layer eye" or ComfyUI "Bypass" mode.

Freeze ❄uses cache

Holds last cooked value. Downstream pulls cache. Useful when upstream changes but you want the proven result.

Bypass ○passthrough

Node deleted from active path. Upstream input bypasses the node, goes direct to downstream. No cache. Re-cooks with each change.

Delete ⨯gone

Node removed permanently. Wire either auto-bridges (P4) or breaks. State lost.

Use cases for bypass over freeze:
  • A/B comparison: bypass to see what the wire produces without a stage
  • Debugging: drop a node from execution without losing it from the graph
  • Cost trim: kill an expensive step temporarily without rewiring
  • Try a Skill that has optional polish nodes you don't want this run
