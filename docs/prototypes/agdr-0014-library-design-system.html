AgDR-0013 forks resolved

F1 Auto-detect vs lock to tools-providers?
YES Auto-detect with prompt-fallback (Layer 3 + 4 hold the contract regardless of provider)
Principle: "Flexibility within constraints." Locking to tool-providers excludes Ollama / LM Studio without benefit — the structural backstop (router gate + validator) holds the LIBRARY-FIRST contract no matter what shape the provider speaks.
F2 Description floor = 60 chars OK?
NO → 80 One full descriptive sentence
Empirical: Speckle ~110, ComfyUI ~95. 80 = floor a search-similarity match needs to discriminate. 60 = guessed; too short.
F3 examples ≥ 1 OK or ≥ 2?
TIER pure ≥1 · host_write ≥2 · network ≥2
Components document STATES. Pure ops have no failure state. host_write needs the offline-host edge (USER-AGENCY demands it). network needs the timeout/auth/quota edge.
F4 Hide create_node_type until search ran?
NO Keep visible; enforce ordering via Layer 3 router gate
Principle: "Consistency over creativity." Conditional tool surfaces are harder to reason about, harder to debug. Structural enforcement via the router gate is cleaner than visibility tricks.
