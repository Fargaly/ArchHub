Cross-host paths — the CAD → Revit litmus

Three architectural paths for the cross-host data-flow problem, all executing the SAME real scenario, side by side. Grounded in n8n AI Agent, LangGraph Plan-and-Execute, Cursor Plan Mode, ComfyUI custom nodes, Houdini CHOPs, Speckle's CAD↔Revit converter, and the Revit API's NewDetailCurve contract.

The honest gap. Today's op set has the endpoints (autocad.read_layer_entities, revit.create_detail_lines) but no adapter between them. The schema, units, and basepoint conventions don't match. def execute(config, inputs, ctx):
    out = []
    for e in inputs["entities"]:
        v = e.get("vertices", [])
        if len(v) < 2:
            continue
        # mm → m, drop z
        p0 = (v[0]["x"]/1000, v[0]["y"]/1000, 0.0)
        p1 = (v[-1]["x"]/1000, v[-1]["y"]/1000, 0.0)
        out.append({"kind":"line","p0":p0,"p1":p1})
    return {"curves": out}

Which path owns which glue?

Different cross-host glue has different best-fit. The three paths are not competitors — they're a layered toolkit. This is what the AI Plan node picks from.

Glue kind Example AI Plan (P1) glue.script (P2) Adapter op (P3) Best fit
Schema coercion field rename, type cast weak — LLM hallucinates names strong — schemas in scope strong — declarative map P2 / P3
Unit conversion mm → m, ft → m forgets edge units trivial 1-liner strongest — Speckle proves it: units as Base prop P3 · bake into the engine
Coordinate-frame transform CAD WCS → Revit PBP LLM doesn't know your basepoint script if matrix given strong — adapter knows Revit basepoints P3

Recommendation if you pick "all three"

The paths layer: P3 is the foundation, P1 is the front door, P2 is the escape valve. Order of implementation:

  1. Ship adapter ops first (P3) — start with geometry.cad_to_revit, geometry.revit_to_cad, data.excel_to_notion. ~1–2 weeks each. They unblock the litmus, they're testable, they give the AI Plan node something canonical to point at.
  2. Ship the ai.plan node (P1) wired to the LIVE op registry as MCP tools/list. Plan output IS the canvas (not a parallel transcript). Approval = the existing Run button. ~2–3 weeks.
  3. Wire hybrid agency via auto_run — per-node flag on ai.plan, default false. Engine forces interrupt-before on any writeable op unless auto_run:true AND no writeable ops in the proposed graph. Borrow Cline's YOLO-mode semantics. ~1 week.
  4. Ship glue.script (P2) as engine-suggested fallback — not a primary entry. Engine raises typed "I can't do this; here are the entities I failed on" → one-click insert glue.script with failed entities pre-filtered + schemas pre-loaded. ~1 week.
  5. Sandbox hardening — RestrictedPython for user-typed scripts; AI-written scripts pass the same gate. ~2 weeks.
  6. Skill dependency-graph warnings in the SaveSkillDialog. ~1 week.

Total: ~9–11 weeks of focused work. The first 1–2 weeks (P3 adapters) close the litmus scenario alone — that's the smallest valuable shippable.
