A library you can read at a glance

Today the node library is 99 flat entries in 15 thin categories, three taxonomies that disagree, near-synonyms (number vs constant, match vs regex_match), and 155 host ops hidden behind raw text config. This prototype regroups everything into 8 families, folds the look-alikes into one node with a mode, tucks every host op under its host, and puts search first. Nothing is deleted — the dead shells become real. Drag from the rail; this is the real dark canvas.

What changed

15 disagreeing categories & 99 flat cards → 8 families. The look-alikes collapse into one node with a mode selector. Host ops live under the host. Three taxonomies become one.

Today (the mess)This prototype
6 INPUT cards — number · text · boolean · file · color · parameter — all the same data.constant → 1 “Value” node with a type switch (number/text/bool/file/color). parameter stays separate (it’s a run-time input, different intent).
16 SHAPE verbs — filter · sort · group_by · reduce · accumulate · dedupe · unique · count · first · last · flatten · pluck · coalesce · ensure · json_codec · join → “Reshape” family: one card per real shape (Filter, Sort, Group, Join, Reduce, Pick) — the rest are op presets inside, not separate cards.
12 MATH cards (add…not_op) + 9 TEXT cards (concat…regex_split) → 1 “Math” + 1 “Text” node, each with an op selector. match/regex_match/regex_* fold into Text·op=regex.
4 AI cards + a hidden ai selector that already unifies them → 1 “AI” node, mode = Chat / Complete / Classify / Tools / Plan. The selector that already exists becomes the visible one.
155 host ops reachable only as raw host+op text on connector.run; 19 hosts flat → Apps & Hosts family — expand a host (Revit, Excel…) to see its ops as a searchable sub-list. Pick the op, it fills connector.run.
8 dead JSX cats (host/read/filter/transform/annotate/compose/connector_op/custom) + 8 unreachable doc.* + 9 hidden legacy primitives → Dead cats removed from the JSX map (no items, no intent). doc.* surfaced under each host as “Open document”. Hidden legacy stays hidden (back-compat) — not shown, not deleted.

One taxonomy, derived not hand-kept

No more grammar-15 vs JSX-23 vs engine-6 disagreeing.
The 8 families are computed from each primitive’s cat by a single FAMILY_OF map. The JSX CAT object is generated from the grammar payload, so it can never grow stale entries again — the “everything renders as NODE” bug class is gone at the root.
