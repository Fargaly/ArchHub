## RAG legend
- 🟢 **REAL** — works end-to-end, a user reaches it, state observably changes.
- 🟡 **PRIMITIVE / DORMANT** — code + tests exist, but no runtime drives it, or it's seeded/synthetic, or it has no UI entry.
- 🔴 **BROKEN / FAKE / DEAD** — contradicts itself, renders wrong, decorative, or actively misleading.

## 1. Desktop app — `app/` core Python  🟢 (mostly real)

- **Boot:** `main.py → WebShell → ArchHubBridge` on a QWebChannel. WebShell is mandatory (raises if QtWebEngine missing — no fallback). Legacy `studio_shell.py` (168 KB) + `workspace_shell.py` (33 KB) are **dead** (no boot path reaches them).
- **bridge.py = 109 JS-callable `@pyqtSlot`s** (the real API surface; module docstring documents only ~16 — stale). 19 Python→JS signals. Off-thread pools (`_cached_async` ×6, `_bg_pool` ×8) + `_cook_lock`; slow slots correctly signal-back (AgDR-0035/0036).
- **Brain integration:** `memory_gate.py` `BrainClient` → daemon `:8473`, 4 hooks inside `llm_router._complete_once`, enrich-not-gate (graceful degrade). `ResilientBrainClientAdapter` = circuit breaker + journal.
- **Cloud integration:** `cloud_client.py` (base `https://cloud.archhub.io`), `cloud_auth.py` (PKCE sign-in), `cloud_sync.py` (GitHub-backed data repo).
- **Risks:** three parallel node/tool registries that must agree (`tool_engine.TOOLS` hand-list + `connectors.base` ~135 ops + `workflows.registry`) — `grammar_health.py` exists to catch the drift; `marketplace_signing.TRUSTED_KEYS["official"]` is a **hardcoded dev placeholder Ed25519 key** ("REPLACE before v1.0"); bare `except: pass` everywhere hides real failures (contradicts honest-status in spots); `first_run.py` checks a provider `"relay"` that exists nowhere.

## 3. personal-brain-mcp — the brain daemon  🟡 (real library, dormant runtime, seeded data)

- **Real:** 22 `brain.*` MCP tools, FastMCP HTTP `:8473`, SQLite+FTS5 BrainStore, genuine algorithm modules (privacy DP, ACL, redaction, reflexion, calibration R1, exploration R2, liveness R3, reputation R4, HLC, multimodal phash+CLIP). 366 tests, algorithms unit-tested for real.
- 🟡 **Store is seeded, not earned:** `brain.db` = 223 fragments / 13 skills, but **every skill is a backfill artifact** (`docs_apply_flow`, etc., 0/0 success counts), 204/223 fragments synthetic `project`-scope from `tools/backfill_skills.py` + `brain_unify.py`. The "204 facts" shown in the in-app brain viz is the **memory graph** (`memory/graph.sqlite`), a *different* store — the daemon brain has 2 organic facts.
- 🔴 **No background worker runs.** `SyncWorker`/`PublishWorker`/`ReflexionWorker`/`Watchdog` are **never started** by the daemon — `build_server` registers tools only. Sync, federation publish, watchdog, async reflexion are dormant.
- 🔴 **skill_mint never reflects:** persists a TRACE + scores gates, but never calls `reflect_on_trace` → **no skill is ever minted from a live trace**. All 13 skills came from the backfill script.
- 🔴 **Calibration loop is open:** `mints_proposed:30, accepted:30, alpha:1.0, beta:1.0` — `record_outcome` never wired, so the "self-tightening" floor never tightens.
- 🟡 **Federation empty + non-durable:** `reputation`/`access_log` = 0 rows, federation server outbox/reputation are in-memory dicts, actor `publicKeyPem` = literal `"PLACEHOLDER"` (ActivityPub sig verify not real), two parallel reputation systems (legacy linear used on the live path, not the R4 Bayesian).
- 🟡 **#33 collective models = export plumbing only.** `dataset_export` + `cloud_archive` + `privatize_for_collective` are real; there is **no training code, no model, no cross-user aggregation**. Consistent with task #33 = pending.
- **Scopes:** 5 in code (USER/PROJECT/FIRM/COMMUNITY/GLOBAL); `privacy.py` aliases COMMUNITY as "collective."

## The honest one-paragraph state

ArchHub is a **genuinely substantial, mostly-real desktop app** with a working graph canvas, 18 real host connectors, a live (if de-clawed) cloud backend, and an unusually rigorous governance system. The **gap is the vision layer**: the "ambient brain" is a high-quality library of primitives whose background runtime never starts and whose store is seeded with synthetic data; the collective-models north star is export plumbing with no model; the cloud's paid value prop (LLM proxy) is switched off; and the UI still carries dead panels, six decorative/no-op buttons, and three fake-telemetry strings — the exact "empty shells and clicks that don't do real work" the founder has complained about. None of this is unfixable, and much of it is small. But the accurate word for the vision layer is **"primitives shipped, runtime pending,"** not "shipped."

## Highest-value fixes (root-cause, by impact)

1. **Start the brain workers + wire skill_mint→reflexion→calibration** — turn the dormant ambient brain on (else the moat is a library, not a brain).
2. **Kill the UI fakes + dead-ends** — remove fake telemetry strings, wire or delete the 6 decorative buttons, fix `LM.paper`/`LM.accent2`, fix the ~10 `bridgeJson`-on-async no-ops, delete the ~1000 lines of orphan panels.
3. **Fix the workflow↔connector op-id contract break** — `host_typed`/`render_typed` reference ops no connector implements.
4. **Reconcile governance bookkeeping** — fix the CLAUDE.md ReactFlow line, flip the 6 drifted AgDRs, sync FAILURE_LOG vs status, drop the dropped Speckle-wire tasks.
5. **Auto-sync the two brains** (or merge them) — remove the manual `brain_unify.py` dependency.
6. **Make the cloud honest** — set `ENV=production`, fix token metering, add token revocation, or clearly mark the proxy as intentionally off.
7. **Real UI tests** — at least CDP smoke tests in CI for the top surfaces, not string-greps.
8. **Wire Procore** (built, unwired) or delete it.
